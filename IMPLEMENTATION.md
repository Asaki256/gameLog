# GameLog - クライアントモック実装

## 実装概要

このプロジェクトは、積みゲー管理アプリ「GameLog」のクライアント側モックアプリケーションです。
バックエンドAPIは実装せず、仮データを使用してフロントエンドの動作を完全に再現しています。

## 技術スタック

- **Next.js 14** (App Router)
- **TypeScript**
- **Tailwind CSS**
- **shadcn/ui** スタイルのコンポーネント
- **Recharts** (グラフ表示)

## 実装した機能

### 1. ダッシュボード (`/`)

- 統計サマリー（未プレイゲーム数、積みゲー総額、今月のクリア数、総ゲーム数）
- プラットフォーム別内訳（プログレスバー表示）
- 次にプレイするゲーム（優先度順、ランダム選択機能付き）
- 最近のアクティビティ

### 2. ゲームライブラリ (`/library`)

- グリッド表示・リスト表示の切り替え
- フィルター機能（プラットフォーム、ステータス）
- ソート機能（購入日、タイトル、プレイ時間、優先度）
- 検索機能
- 12件のモックゲームデータ

### 3. ゲーム詳細 (`/library/[id]`)

- ゲーム基本情報の表示
- ステータス・優先度・メモの編集
- プレイ履歴の表示

### 4. ゲーム登録 (`/library/new`)

- IGDB API風の検索機能（モック）
- フォーム入力（タイトル、プラットフォーム、価格、購入日、優先度、メモ）

### 5. 統計 (`/stats`)

- 月別クリア本数グラフ（棒グラフ）
- 月別プレイ時間グラフ（折れ線グラフ）
- プラットフォーム別内訳（円グラフ）
- ステータス別内訳（横棒グラフ）
- プレイ時間トップ5

### 6. Wishlist (`/wishlist`)

- 購入予定ゲーム一覧
- 積みゲー警告（5本以上で表示）
- Wishlistサマリー（合計アイテム数、予想総額、発売予定数）

### 7. 設定 (`/settings`)

- プロフィール編集
- Steam連携機能（モック）
  - Steam ID入力
  - ライブラリ取得（モックデータ）
  - ゲーム選択インポート
- 月間目標設定
- データエクスポート（モック）
- アカウント削除（モック）

## モックデータ

### ゲームデータ (`lib/mockData.ts`)

- 12件のゲームデータ
  - Elden Ring、Zelda、Persona 5 Royal、Cyberpunk 2077など
  - 各ゲームに完全なメタデータ（価格、購入日、プレイ時間、優先度など）

### Wishlistデータ

- 3件の購入予定ゲーム

### 統計データ

- 過去6ヶ月分の月別統計データ

### Steam連携モックデータ

- 5件のSteamゲームデータ

## 起動方法

### 開発サーバー

```bash
npm install
npm run dev
```

ブラウザで http://localhost:3000 を開く

### 本番ビルド

```bash
npm run build
npm start
```

## 主な特徴

### 1. 完全に動作するモックアプリケーション

- すべての機能が実際に動作
- バックエンドAPIなしでフロントエンドの動きを完全に再現
- フィルター、ソート、検索などのインタラクティブ機能が実装済み

### 2. 仕様書に忠実な実装

- README.mdの仕様書に記載された機能をほぼすべて実装
- プラットフォーム、ステータス、優先度などのデータモデルも仕様通り

### 3. モダンなUI/UX

- shadcn/uiスタイルのコンポーネント
- レスポンシブデザイン対応
- Tailwind CSSによる美しいスタイリング

### 4. TypeScriptによる型安全性

- すべてのデータモデルに型定義
- 型安全なコンポーネント実装
